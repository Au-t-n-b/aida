# AIDA 数据中心对外接口设计

> 本文档从《数据中心设计-v6》中摘取，聚焦数据中心对外提供的核心接口，覆盖四组能力：
> **组织资产、项目资产、项目管理、配置中心-人员管理（用户管理）**。
> 主要说明这些接口"有哪些、分别是什么功能"，供前端、Agent、外部系统对接使用。

---

## 1 范围与对接说明

| 能力组 | 说明 | 典型对接方 |
| --- | --- | --- |
| 组织资产 | 跨项目的只读组织资产文件（`org-assets`）的列表与目录树 | 前端「组织资产」页、外部系统引用 |
| 项目资产 | 单个项目内业务文件的目录树、目录增删、文件列表/上传/下载/删除 | 前端「项目资产」页、Agent 产物回传 |
| 项目管理 | 项目的新建/列表/详情/修改，以及审批/驳回/重新提交/待审批列表/归档 | 前端「项目管理」页、外部系统建项目 |
| 配置中心-人员管理 | 用户（人员）的列表/详情/新建/修改/删除 | 前端「配置中心-用户管理」页（仅管理员） |

> **调用方标注**：
> - `外部`：数据交换接口，人/机器皆可调用，**可选鉴权**——带 token 走 RBAC，不带 token（机机直调）本期放行。
> - `内部`：管理类接口，仅控制台/人调用，**强制 JWT + RBAC**，无 token 返回 `401`。

---

## 2 通用约定

- **路径前缀**：所有接口以 `/api/v1` 开头。
- **项目类接口 path 参数**：`{projectId}` 取 `projects.project_id`（UUID32 去横线），**非**内部数字 `id`。
- **统一响应包络**：

```json
{
  "code": 0,
  "message": "success",
  "data": { }
}
```

- **分页约定**：列表类入参 `page`（默认 1）、`pageSize`（默认 20）；出参 `data` 含 `list` 与 `total`。
- **时间格式**：ISO 8601。
- **鉴权头**：内部接口需 `Authorization: Bearer {token}`。

## 4 组织资产接口

跨项目的只读组织资产区（物理目录 `{BUSINESS_ROOT}/org-assets/`），前端「组织资产」页展示。仅管理员可上传/修改/删除，其余只读。

### 4.1 组织资产列表 `GET /org-assets`

| 项 | 内容 |
| --- | --- |
| 功能 | 列出组织资产下所有文件 |
| 入参（query） | `keyword` 可选（按名称模糊） |
| 出参（data） | `list[]`：`fileId`、`fileLogicalName`、`ext`、`sizeBytes`、`updatedAt` |

### 4.2 组织资产目录树 `GET /org-assets/tree`

| 项 | 内容 |
| --- | --- |
| 功能 | 返回组织资产（`project_id=NULL`）的目录/文件节点树，供前端按目录浏览 |
| 入参 | 无 |
| 出参（data） | `tree`：嵌套节点 `nodeId`/`name`/`logicalPath`/`type`(dir/file)/`fileId`(文件节点)/`children[]`；首次访问自动确保 org-assets 根节点存在 |

## 5 项目资产接口

单个项目内业务文件的目录与文件操作，前端「项目资产」页展示。目录/文件层级以 `fs_nodes` 表（邻接表）为权威来源，目录树查库渲染、不扫盘。

### 5.1 项目目录树 `GET /projects/{projectId}/tree`（内部）

| 项 | 内容 |
| --- | --- |
| 功能 | 返回该项目的目录/文件树（模板骨架 + 用户新建目录 + 已上传文件），供选定上传位置 |
| 入参 | path `projectId`（UUID32）；query `includeFileCount` 可选 |
| 出参（data） | `tree`：节点 `nodeId`/`name`/`logicalPath`/`type`(dir/file)/`moduleCode`/`fileStage`/`phase`(本期/暂不)/`isSystem`/`fileId`/`children[]` |
| 业务规则 | 需登录；"暂不"模块节点标 `phase=暂不`，访问其内容返回"未开放" |

### 5.2 新建目录 `POST /directories`

| 项 | 内容 |
| --- | --- |
| 功能 | 在指定父目录下新建子目录（用户目录，可删） |
| 入参（body） | `parentNodeId` 必填（父目录 `fs_nodes.id`）、`name` 必填 |
| 出参（data） | `nodeId`、`logicalPath` |
| 业务规则 | `is_system=false`；继承父节点 `projectId`/`moduleCode`/`fileStage`/`phase`；磁盘同步 `mkdir`；同级重名返回 `400` |

### 5.3 删除目录 `DELETE /directories/{nodeId}`

| 项 | 内容 |
| --- | --- |
| 功能 | 删除用户新建目录及其下全部内容 |
| 入参（path） | `nodeId` |
| 出参（data） | `deleted`=true |
| 业务规则 | `is_system=true` 模板目录禁删返回 `403`；用户目录**级联删除**其下所有子目录+文件（含物理文件/目录，不可恢复） |

### 5.4 文件列表 `GET /files`

| 项 | 内容 |
| --- | --- |
| 功能 | 按项目/模块/阶段筛选文件 |
| 入参（query） | `projectId` 可选（空=组织资产）；`moduleCode` 可选；`fileStage` 可选；`keyword` 可选；`page`；`pageSize` |
| 出参（data） | `list[]`：`fileId`、`fileLogicalName`、`fileName`、`logicalPath`、`ext`、`sizeBytes`、`version`、`source`、`uploaderId`、`createdAt`；`total` |

### 5.5 上传文件 `POST /files/upload`

| 项 | 内容 |
| --- | --- |
| 功能 | 流式上传文件到指定目标目录，并登记元数据（同时建 `fs_nodes` 文件节点） |
| 入参（multipart/form-data） | `file` 必填；`targetNodeId` 必填（目标目录的 `fs_nodes.id`；组织资产传 org-assets 根节点）；`fileLogicalName` 可选 |
| 出参（data） | `fileId`、`nodeId`、`fileName`、`logicalPath`、`moduleCode`、`fileStage`、`sizeBytes`、`checksum` |
| 业务规则 | `projectId`/`moduleCode`/`fileStage`/`logicalPath` 由 `targetNodeId` 节点直接读取；目标非目录或不存在返回 `400`；目标在"暂不"模块返回"未开放"；超大小上限返回 `3002`；同名按传入文件名覆盖（复用同一文件节点） |

### 5.6 下载文件 `GET /files/{fileId}/download`

| 项 | 内容 |
| --- | --- |
| 功能 | 流式下载文件本体 |
| 入参（path） | `fileId` |
| 出参 | 二进制流，带 `Content-Disposition` 文件名 |
| 业务规则 | 文件不存在返回 `3001` |

### 5.7 删除文件 `DELETE /files/{fileId}`

| 项 | 内容 |
| --- | --- |
| 功能 | 物理删除文件（不可恢复） |
| 入参（path） | `fileId` |
| 出参（data） | `deleted`=true |
| 业务规则 | 删物理文件 + 删 `files` 行 + 删对应 `fs_nodes` 文件节点；文件不存在返回 `3001` |

## 6 项目管理接口

项目的全生命周期管理：新建即「待审批」，仅管理员审批；只有「已审批」项目可正式使用；项目不可物理删除，仅软归档。物理目录名用 `project_id`（UUID32）。

### 6.1 新建项目 `POST /projects`

| 项 | 内容 |
| --- | --- |
| 功能 | 系统生成 `projectId`（UUID32），写项目表，一次性按模板建好"本期"模块完整空目录骨架（"暂不"模块仅建占位顶层目录）；需登录，创建人取自当前用户 |
| 入参（body） | `projectName` 必填；`bidCode`/`customerName`/`projectCode` 可选；`tdUserId`/`pdUserId`/`pcmUserId` 可选 |
| 出参（data） | `id`、`projectId`（UUID32）、`status`(=PENDING_APPROVAL)、`rootPath` |
| 业务规则 | 项目名查重，重名返回 `2002` |

### 6.2 项目列表 `GET /projects`

| 项 | 内容 |
| --- | --- |
| 功能 | 分页查询项目，支持状态/关键字筛选 |
| 入参（query） | `status` 可选；`keyword` 可选（项目名模糊）；`page`；`pageSize` |
| 出参（data） | `list[]`：`id`、`projectId`、`projectName`、`status`、`creatorId`、`createdAt`；`total` |

### 6.3 项目详情 `GET /projects/{projectId}`

| 项 | 内容 |
| --- | --- |
| 功能 | 查询单项目全字段及成员 |
| 入参（path） | `projectId`（UUID32） |
| 出参（data） | 项目全字段 + 成员角色列表 `members[]`（userId/roleCode/roleName） |

### 6.4 修改项目 `PUT /projects/{projectId}`

| 项 | 内容 |
| --- | --- |
| 功能 | 修改项目基本信息 |
| 入参 | path `projectId`；body 可改 `projectName`、`tdUserId`、`pdUserId`、`pcmUserId` |
| 出参（data） | 更新后的项目对象 |
| 业务规则 | 不提供删除接口；归档通过状态流转实现 |

### 6.5 项目审批 / 驳回 / 重新提交 / 待审批 / 归档（内部）

项目状态机：`待审批 →(approve) 已审批`、`待审批 →(reject) 已驳回`、`已驳回 →(resubmit) 待审批`、`已审批 →(archive) 已归档`。

| 接口 | 入参 | 出参 / 规则 |
| --- | --- | --- |
| 审批通过 `POST /projects/{projectId}/approve` | path `projectId`；body `approverId` 必填、`comment` 可选 | `status`=APPROVED；需 `project:approve`；仅 PENDING_APPROVAL 可审批，否则 `2001`；写审计 |
| 审批驳回 `POST /projects/{projectId}/reject` | path `projectId`；body `approverId` 必填、`comment` 必填 | `status`=REJECTED；需 `project:approve`；仅 PENDING_APPROVAL 可驳回，否则 `2001`；写审计 |
| 重新提交 `POST /projects/{projectId}/resubmit` | path `projectId`；无 body（操作者取自 JWT） | `status`=PENDING_APPROVAL；仅 REJECTED 可重新提交，否则 `2001`；仅需登录；写审计 |
| 待审批列表 `GET /projects/pending-approval` | query `page`、`pageSize` | `list[]`：id/projectId/projectName/creatorId/creatorName/createdAt；`total`；需 `project:approve` |
| 归档（占位）`POST /projects/{projectId}/archive` | path `projectId`；body `operatorId` 必填、`comment` 可选 | `status`=ARCHIVED；需 `project:archive`；仅 APPROVED 可归档，否则 `2001`；写审计 |

## 7 配置中心-人员管理接口

用户（人员）的 CRUD，前端「配置中心-用户管理」页使用。全部走全局 RBAC（`require_permission`），默认仅管理员（ADMIN）拥有 `user:*` 权限。

### 7.1 用户列表 `GET /user-mgmt`

| 项 | 内容 |
| --- | --- |
| 权限 | `user:list` |
| 入参（query） | `keyword` 可选；`status`(int) 可选；`page`；`pageSize` |
| 出参（data） | `list[]`：`userId`、`username`、`email`、`status`、`createdAt`、`updatedAt`；`total` |

### 7.2 用户详情 `GET /user-mgmt/{userId}`

| 项 | 内容 |
| --- | --- |
| 权限 | `user:read` |
| 入参（path） | `userId` |
| 出参（data） | 用户对象（同上字段） |
| 业务规则 | 用户不存在返回 `1004` |

### 7.3 新建用户 `POST /user-mgmt`

| 项 | 内容 |
| --- | --- |
| 权限 | `user:create` |
| 入参（body） | `username` 必填；`password` 必填；`email` 可选；`status` 可选（默认 1） |
| 出参（data） | 用户对象 |
| 业务规则 | 用户名已存在返回 `1001` |

### 7.4 修改用户 `PUT /user-mgmt/{userId}`

| 项 | 内容 |
| --- | --- |
| 权限 | `user:update` |
| 入参 | path `userId`；body `email` 可选、`status` 可选 |
| 出参（data） | 更新后的用户对象 |
| 业务规则 | **不可修改 username / password**；用户不存在返回 `1004` |

### 7.5 删除用户 `DELETE /user-mgmt/{userId}`

| 项 | 内容 |
| --- | --- |
| 权限 | `user:delete` |
| 入参（path） | `userId` |
| 出参（data） | `deleted`=true |
| 业务规则 | 删除自身账号返回 `1005`；用户不存在返回 `1004` |

### 7.6 删除用户（认证模块入口）`DELETE /users/{userId}`

| 项 | 内容 |
| --- | --- |
| 权限 | `user:delete` |
| 入参（path） | `userId` |
| 出参（data） | `deleted`=true |
| 业务规则 | 与 §7.5 逻辑一致：删除自身 `1005`、用户不存在 `1004`（两个入口，便于不同前端路径调用） |

## 8 错误码（本文档相关）

| code | 含义 | 说明 |
| --- | --- | --- |
| 0 | 成功 | — |
| 400 | 参数错误 | 入参校验失败、目标节点非法、同级重名等 |
| 401 | 未认证 | 缺少/无效 Token（内部接口） |
| 403 | 无权限 | RBAC 校验未通过 / 模板目录禁删 |
| 404 | 资源不存在 | 项目等不存在 |
| 500 | 服务内部错误 | — |
| 1001 | 用户名已存在 | 新建用户查重 |
| 1004 | 用户不存在 | 用户管理 get/update/delete |
| 1005 | 不能删除自身账号 | 删除用户 |
| 2001 | 项目状态不允许该操作 | 审批/驳回/重新提交/归档非法状态 |
| 2002 | 项目名称已存在 | 新建项目查重 |
| 3001 | 文件不存在 | 下载/删除文件 |
| 3002 | 文件上传失败 | 存储写入异常 / 超大小上限 |
