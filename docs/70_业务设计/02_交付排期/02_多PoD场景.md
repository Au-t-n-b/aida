# 场景总述
线下安装，主要是以 PoD 为粒度进行实施的

## 多POD计划排期
支持灵活选择活动节点作为里程碑节点，默认将一些关键节点活动（机房实施准备、到货、上电和集群性调优）作为里程碑

## POD 分批排期
由于大项目下，需要交付多个 PoD。受施工队伍的资源限制，没法一起完成，所以会分批次进行完成。
用户可以对里程碑进行分批次实现，把不同 POD 按照编号，拖到框中合并在一起作为一个批次。比如：一个机房4个POD，每行2个pod是一批等。

## 第一个里程碑调整场景
用户可以对第一个里程碑进行分批，把左侧不同 POD 拖入对应批次。第一个里程碑的分配会作为后续里程碑的默认分批，后续里程碑也支持灵活批次再调整。

## 筛选与定位
支持按机房筛选、搜索 POD / 管理单元、只看已修改或缺失的 POD；也可以跳转到孪生视图查看选中的 POD。

## 沙箱推演
保存批次后，不会立即写入系统，而是先做预验证和倒排沙箱推演，展示这些 POD 活动完成日期会引起多少计划行日期变更。沙箱中运行，把结果给用户看。

## 人工确认下发
推演完成后，倒排面板会出现“确认下发 / 放弃推演”。确认后才会写回 POD 级的里程碑，并同步写入计划排期的计划日期变更。
