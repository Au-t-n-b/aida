# mailgw 邮件网关 · 部署与配置手册

> **读者**：负责部署和运维 mailgw 的工程师
> **版本**：v1.0（2026-06-10）

## 1. 环境要求

- 操作系统：Windows / Linux 均可
- Python 3.11 及以上（开发与测试基于 3.12）
- 可访问公司邮箱的 SMTP 与 POP3 端口（常见为 465/995，以邮箱服务商说明为准）

## 2. 安装步骤

```bash
# 1. 获取代码（复制项目目录或 git clone）
cd mailgw 项目根目录

# 2. 安装依赖
pip install -r requirements.txt

# 3. 准备配置
copy config.yaml.example config.yaml     # Linux 用 cp
copy .env.example .env
# 编辑 config.yaml 与 .env，按下文逐项填写
```

## 3. 配置项说明

### 3.1 config.yaml（非敏感配置，可入版本库）

| 配置项 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `smtp.host` | 字符串 | — | SMTP 服务器地址，如 `smtp.corp.com` |
| `smtp.port` | 整数 | — | SMTP 端口，SSL 直连常见 465 |
| `smtp.ssl` | 布尔 | — | `true`=SMTP_SSL 直连；`false`=明文连接后尝试 STARTTLS（仅当服务器支持时升级） |
| `smtp.username` | 字符串 | — | 登录账号，通常即邮箱地址 |
| `smtp.from_addr` | 字符串 | — | 发件人地址 |
| `smtp.display_name` | 字符串 | `""` | 发件人显示名，如 `AIDA 智能助手` |
| `pop3.host` | 字符串 | — | POP3 服务器地址 |
| `pop3.port` | 整数 | — | POP3 端口，SSL 常见 995 |
| `pop3.ssl` | 布尔 | — | 是否 SSL 连接 |
| `pop3.username` | 字符串 | — | 登录账号 |
| `pop3.poll_interval` | 整数 | `0` | 后台定时拉取间隔（秒）；`0` 表示仅在 agent 调 `list_inbox?refresh=true` 时按需拉取 |
| `policy.whitelist_domains` | 列表 | `[]` | 域名级白名单，如 `["corp.com"]`，命中该域名的地址直发 |
| `policy.whitelist_addresses` | 列表 | `[]` | 精确地址白名单，如 `["vip@other.com"]` |
| `policy.hourly_limit` | 整数 | `20` | 每小时发送上限（封） |
| `policy.daily_limit` | 整数 | `100` | 每日发送上限（封） |
| `policy.max_attachment_mb` | 整数 | `25` | 单封附件总大小上限（MB） |
| `policy.max_recipients` | 整数 | `20` | 单封收件人（to+cc）数量上限 |
| `data_dir` | 字符串 | `./data` | 数据目录（SQLite 库与附件存放处） |

**白名单规则**：一封邮件的 to+cc **全部命中**白名单才直接发出；任何一个地址未命中，整封进入待审批队列，由人工在审批页处理。

### 3.2 .env（敏感信息，禁止提交版本库）

| 变量 | 说明 |
|---|---|
| `MAILGW_SMTP_PASSWORD` | SMTP 授权码（注意：多数企业邮箱需要专门生成的"客户端授权码"，不是网页登录密码） |
| `MAILGW_POP3_PASSWORD` | POP3 授权码（通常与 SMTP 相同） |
| `MAILGW_ADMIN_PASSWORD` | 审批页登录口令（用户名固定为 `admin`） |
| `MAILGW_TOKEN_<调用方名>` | 调用方 API token，可配多个；变量名后缀（转小写）即审计日志中的调用方名。例：`MAILGW_TOKEN_AIDA=随机串` → 审计中记为 `aida` |

token 生成建议：`python -c "import secrets; print(secrets.token_urlsafe(32))"`

## 4. 公司邮箱侧准备

1. 在邮箱设置中**开通 SMTP / POP3 服务**，并生成客户端授权码；
2. **注意 POP3 可见范围**：部分企业邮箱（如腾讯企业邮）POP3 默认只暴露"收件箱"或"最近 30 天"的邮件，如 agent 需要读取更早的来信，请在邮箱设置中调整收取范围；
3. mailgw 拉取邮件后**不会删除**服务器上的邮件，邮件保留策略以邮箱侧设置为准。

## 5. 启动与验证

```bash
python -m mailgw --config config.yaml --env .env --host 127.0.0.1 --port 8025
```

验证 API（PowerShell）：

```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:8025/api/inbox?limit=1" `
  -Headers @{ Authorization = "Bearer <你的token>" }
```

验证审批页：浏览器打开 `http://127.0.0.1:8025/admin`，输入 `admin` / 你设置的口令。

> 默认仅监听 127.0.0.1（与 AIDA 同机部署）。如需局域网访问审批页，用 `--host 0.0.0.0` 并确保防火墙策略受控。

## 6. 数据与备份

```
data/
├── mailgw.db           # SQLite：发件队列、收件缓存、审计日志
└── attachments/<邮件id>/<文件名>   # 收到的附件
```

备份建议：定期复制整个 `data/` 目录（先停服务或接受秒级不一致）。审计记录在 `audit_log` 表，可用任意 SQLite 工具查询。

## 7. 升级

替换代码后重启即可。数据库 schema 使用 `CREATE TABLE IF NOT EXISTS`，已有数据向后兼容；涉及表结构变更的版本会在变更说明中单独给出迁移步骤。

## 8. 常见问题

| 现象 | 原因与处理 |
|---|---|
| 发送报 535 认证失败 | 授权码错误或未开通 SMTP；确认填的是**授权码**而非登录密码 |
| 连接超时 | 防火墙拦截 465/995 端口，或 host/port 配错 |
| POP3 拉不到某些邮件 | 邮箱侧 POP3 收取范围限制（见 §4），或该邮件已被本地记录（UIDL 去重不会重复拉取） |
| agent 调用返回 401 | token 不在 `.env` 的 `MAILGW_TOKEN_*` 中，或 `Authorization: Bearer` 头格式不对 |
| 发送返回 rejected 且提示"上限" | 触发 `hourly_limit`/`daily_limit` 限流，等待窗口期或调大配置后重启 |
