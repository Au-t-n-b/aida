# skill2langgraph 实现路线图

> 状态：Draft v1（2026-06-08）
> 定位：从**设计**转入**实现**的总纲——回答"最终形态是什么 / 要建什么 / 不建什么 / 按什么顺序建"。
> 关联：`测试底座对齐方案.md`、`引导编写流程-完整设计.md`、`requirements/`（测试底座 / Toolbox / SDUI / 文档库）。

---

## 0. 一句话

最终形态**不是一份指导文档，也不是单一 skill**，而是：**一份机器可校验的契约（核心 IP）+ 一套驱动 coding agent 的隔离编排（编译）+ 一层薄测试底座驱动 + 一个独立前门程序**。其中"编译"由 agent 驱动、几乎零生成代码；真正要建的只有**契约**与**编排策略**，subagent 原语用工具自带。

---

## 1. 最终形态（组件 × 形态 × 现在做成什么 × 谁驱动）

| 组件 | 最终形态 | 现在先做成 | 谁驱动 |
|---|---|---|---|
| **① 契约 / 规则集** | 机器可校验 spec + lint（SkillIR schema · §21+一致性规则 · skill 打包契约 · assumptions schema · 交付契约） | 同左（核心 IP，先冻结） | — |
| **② 编译** | **多-subagent 隔离编排**：会话 A–G 各一个隔离子代理，DAG + 盲化，BuildAgent 适配器（agent-中立） | **一段编排脚本 + 契约**，用单一工具原生 subagent | coding agent |
| **③ 测试底座驱动** | 薄代码：注入 + 在真实 AIDA 跑 + 取证 + 三类验证门 | 最小可用薄驱动（进程态、live） | 代码 |
| **④ 前门** | 独立对话程序（自带模型，业务专家侧） | **先做成 skill 原型**（面谈 + HARD-GATE） | LLM |

**核心洞察（决定"要建什么"）**：
- **build 是 agent 驱动的**——确定性来自被生成并验证的 LangGraph，不是生成它的 agent。所以"编译"无需写生成器代码。
- **subagent 原语是现成的**（Claude Code Agent/Workflow、`claude -p`、`codex exec`、`cursor-agent -p`）——**不造 subagent 运行时**。
- 真正要建的只有 **① 契约** 和 **② 的隔离编排策略**（一段脚本：哪个会话拿哪些输入、按什么顺序、盲化约束）。

## 2. 编译子系统 = 多 subagent 隔离编排

### 2.1 隔离 DAG + 盲化（验证完整性的根基）

```text
Raw Skill ──┬──► [A] SkillIR
            └──► [B] 行为测试            A ⊥ B（都只读 Raw Skill，互不可见）
                      │
        SkillIR ──────┼──► [C] Graph(skill 模块: skill.py+steps+sdui.py+新工具)
                      └──► [D] 契约测试   C ⊥ D（D 只认 SkillIR，对 Graph 盲）
                      │
              [测试底座：代码执行 Graph on 真实 AIDA]   ← 非 agent
                      │
            ┌──► [E] Graph 文档  [F] 行为报告  [G] 契约报告   （闸门后）
```

盲化是**对抗性独立**：同一 agent 既写 Graph 又写其测试 = 自己批改自己。故 **A⊥B、C⊥D 必须强制**——由父编排"喂给每个子代理的输入清单"保证。

### 2.2 subagent 原语 vs 隔离策略（谁建）

| | 谁提供 | 建不建 |
|---|---|---|
| subagent 原语（隔离上下文的子代理） | 工具**原生** | 不建，直接用 |
| 隔离策略（DAG + 盲化输入清单 + 文件式产物 + 验证门） | 没人替你做 | **建**，但是**编排脚本/prompt，非运行时** |

### 2.3 agent-中立：一个可延后的决策

- **现在·锁定一家**（如 Claude Code）：直接用其原生 Agent/Workflow 起 A–G 子代理，② ≈ **一段 Workflow 脚本 + 契约**，零额外建设。
- **将来·agent-中立**：三家原生 subagent 机制不可移植 → 退到 headless 单发（`claude -p`/`codex exec`/`cursor-agent -p`，一次调用=一个隔离会话）+ 适配器统一文件式产物。
- agent-中立**安全**（下游验证门 agent-中立，谁生成都过同一套测试），但是**可延后的独立投资**，非 MVP 必需。
- **可升级要求**：把 session 边界、文件式产物、盲化输入清单写干净，将来只需补适配器、编排策略不动。

## 3. 要建的 / 不建的

### 要建
| 类 | 内容 | 形态 |
|---|---|---|
| ① 契约 | SkillIR schema · §21完整性+一致性规则（S8 与 Session 0 **同源**）· skill 打包契约（BaseSkill/steps/sdui/SKILL.md/state.json schema）· assumptions schema（含 `id↔Graph 回溯`）· 交付契约 | spec + lint |
| ② 编排 | A–G 隔离 DAG + 盲化输入清单 + 文件式产物 + 三类验证门 | 编排脚本 |
| ③ 测试底座驱动 | 注入 skill → 真实 AIDA 跑 → 读 `state.json`/checkpoint/Langfuse → 三类验证 | 薄代码 |
| ④ 前门 | S1–S10 面谈 + HARD-GATE，产出 `SKILL.md`+`assumptions.json` | skill 原型 → 独立程序 |
| 回放服务 | LLM + 外部工具端点回放（确定性） | 薄代码（P1 可后补，先 live） |

### 明确不建
- ❌ **subagent 运行时**（用工具原生）。
- ❌ **AIDA 复刻 Harness**（用真实 AIDA 本体）。
- ❌ **大代码生成器**（生成由 agent 驱动）。
- ❌ **数据中心里的 run/skill 存储**（run 态→容器、skill 代码→镜像、skill 注册→数据中心 Skill 中心、项目文件→数据中心 files API）。

## 4. 实现顺序

| Step | 交付物 | 验收 | 形态 |
|---|---|---|---|
| **1 冻结契约** | 全部 ✅草案（`contracts/`）：SkillIR-schema · 校验规则集 · assumptions-schema · skill打包契约 · 交付契约 | 能对一份手写 Raw Skill 跑通校验、对一个 skill 包跑通 lint | spec + lint |
| **2 编译编排** | A–G 隔离编排脚本（DAG+盲化+文件契约），先用一家原生 subagent | 给定 Raw Skill，自动产出 SkillIR + skill 模块 + 行为/契约测试，且 B⊥A、D⊥C | 编排脚本 |
| **3 最小测试底座驱动** | 注入 `aida/agent/skills` + live 跑 + 取证 + 三类验证 | 一个生成的 skill 在真实 `aida/agent` 进程跑完、三类验证可判定 | 薄代码 |
| **4 Walking skeleton** | 一条端到端竖切：手写 Raw Skill → 编译 → 真实 AIDA 跑 → 三类验证全绿 | 范式被证明；真问题暴露 | 组合 1–3 |
| **5 前门** | S1–S10 skill 原型（复用 Step 1 规则集做 S8 HARD-GATE）→ 独立程序 | 业务描述 → 合规 `SKILL.md`+`assumptions.json` | skill → 程序 |

**为什么这个顺序**：契约是 ②③④ 共同地基（先）；②③ 先于 ④（契约允许手写 Raw Skill，不必等前门）；Step 4 是**验证**而非新建（跑通即证明 ①②③）。

## 5. 分阶段（对齐 AIDA 成熟度 P0→P3）

| 阶段 | AIDA 现状 | 本范式可做 |
|---|---|---|
| **P0（现在·单机无容器）** | `aida/agent` 进程可跑、live LLM 可用 | Step 1–4 全部可做：进程态注入、live 跑、commit 锁版本、本地回放 |
| **P1** | 数据中心 API/OpenAPI、文档库孪生 | 真实能力接地（采 `/openapi.json`）、回放服务、前门 |
| **P2** | Manager + 容器 + Postgres | 一次性容器、**镜像 digest 钉死**、测试租户、Toolbox 注册回流、Skill 中心发布（draft→published） |
| **P3** | 多租户事件 | 再验证流水线（新 digest 批量重验）、agent-中立适配器全自动 |

## 7. 实现期不可违反的设计约束

1. **零静默决策**：每处"模糊→确定"进 `assumptions.json` 并经作者确认；编译期不替业务拍板。
2. **盲化独立**：A⊥B、C⊥D；测试绝不迁就实现。
3. **真身测试**：在真实 AIDA 跑，不造复刻；*测试通过 ≡ 在 AIDA 上通过*。
4. **存储分工**：run 态→容器；skill 代码→镜像、注册→数据中心 Skill 中心；项目文件→数据中心 files API。
5. **工具可增长 / SDUI 封闭**：工具复用>自实现>注册回流；SDUI 复用-only，缺则报缺口。
6. **版本钉死**：对齐 ≡ 测试镜像 digest == 生产镜像 digest（P2 起）。
