## 2. 一个 skill 牵扯到哪些文件？⭐（先看懂这张，再读细则）

> 新人最容易懵的地方：**一个 skill 不在一个文件里，它纵向被切成"范式→手册→代码"，横向散在几个目录。** 看懂这张图，再去 §3/§4 读细则就不会迷路。

```
            想做 zhgk 这样一个业务场景 Skill
                       │
   ┌───────────────────┼────────────────────┐
   │ 为什么(范式)       │ 怎么做(手册·全在 docs/) │ 实物(代码·不在 docs/)
   ▼                   ▼                     ▼
20_架构与范式/03_范式   30_skill开发/           skills/zhgk/SKILL.md      ← A 层(声明/触发器)
                       31_手写规范/SKILL-DEV     skills/zhgk/references/*  ← 分步详规
                       31_手写规范/TOOL-DEV      agent/skills/zhgk/        ← B 层(真实执行)
                       31_手写规范/SDUI          ~/.claude/skills/zhgk/    ← A 层部署副本
                       31_手写规范/接入/         
40_评测/METRICS        30_skill开发/样板盘点_zhgk(验收基准)
```

**两层契约**：A 层 `SKILL.md` 的「后端节点」表 ↔ B 层 `steps/*.py` 的 `step.key`，由 `lint_skill_contract` 自动卡死，改一层不改另一层就提交失败。
**为什么 A/B 分层**：A 层是给 AI 看的"说明书+触发器"（读它就知道该不该召唤这个 skill，不用读代码）；B 层是机器跑的实现。
**两条造 skill 的路线**：手写走 [`31_手写规范/`](30_skill开发/31_手写规范/README.md)（当前可用）；自动编译走 [`39_编译路线_skill2langgraph/`](30_skill开发/39_编译路线_skill2langgraph/README.md)（skill2langgraph · 并行开发中）。
