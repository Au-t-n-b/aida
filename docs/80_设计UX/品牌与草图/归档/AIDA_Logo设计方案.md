# AIDA Logo 设计方案

## 设计对象

- 系统名：`AIDA`
- 发音：艾达
- 英文全称：`Agentic Integrated Delivery Arena`
- 中文语义：智能体集成交付架构
- 核心判断：AIDA 不是一个轻量助手名，而是从传统 `ISDP / Integrated Delivery` 平滑升级到 `Agentic Delivery Architecture` 的新一代交付系统品牌。

## 品牌定位

AIDA 的 Logo 应该同时传达四件事：

1. `Agentic`：系统具备主动理解任务、调动工具、推进闭环的智能体能力。
2. `Integrated`：系统是多平台、多工具、多角色的集成中枢，而不是单点功能入口。
3. `Delivery`：系统最终指向交付执行、作业推进、现场闭环和结果达成。
4. `Architecture`：系统是企业级底座，强调稳定、结构、可信和可扩展。

因此视觉气质不建议做成炫酷 AI 图标、机器人头像、赛博渐变或泛科技电路板，而应接近“企业级智能交付架构品牌”：冷静、精确、有力量、可长期使用。

## 主推方案：AIDA Gateway

### 设计命名

`AIDA Gateway / 智能交付之门`

### 核心图形

以大写字母 `A` 为基础，构造一个“架构之门”：

- 两侧斜向结构形成稳定的建筑框架，对应 `Architecture`。
- 中央负形通道形成向上的交付路径，对应 `Delivery`。
- 横向红色连接梁贯穿左右结构，对应 `Integrated`。
- 红色节点与路径线表达智能体调度、任务编排和主动执行，对应 `Agentic`。

这个方向的优势是：不用解释也能读出 `A`，同时可以把“架构、集成、交付、智能体”四层语义压缩进一个强识别图形。

### 字母语义映射

| 字母 | 视觉表达 | 业务含义 |
| --- | --- | --- |
| A | 上升的结构门形 | Agentic，从信息化升级为智能体化 |
| I | 横向连接梁与中心通道 | Integrated，多平台、多工具统一接入 |
| D | 向前推进的红色路径 | Delivery，任务承接、执行、闭环 |
| A | 稳定的几何框架 | Architecture，企业级底座和长期架构 |

## 备选方向

### 方向 2：Integrated Delta

用三角形 / Delta 表达“变化、升级、推进”，内部用三条线路汇聚到一个执行节点。适合强调“从旧平台到新架构的跃迁”，但品牌记忆点会弱于直接使用 `A`。

### 方向 3：Agentic Hub

用中心节点和四个外围节点表达智能体编排网络。适合强调中枢、调度、集成，但容易落入常见 SaaS / AI 平台图标套路，需要非常克制。

### 方向 4：Architecture Grid

用工程蓝图、模块网格和交付路径组成标志。适合 PPT 与咨询汇报场景，但作为 Logo 的独立识别度不足，更适合作为辅助图形系统。

## 推荐结论

主 Logo 采用 `AIDA Gateway`。它最符合 AIDA 当前的命名逻辑：既保留 `Integrated Delivery` 的历史传承，又把前缀升级为 `Agentic`。该方案也最容易做成企业内外都能理解的标志系统。

## 视觉系统

### 主标

- 图形：`A` 字母门形 + 红色集成路径
- 字标：`AIDA`
- 辅助英文：`Agentic Integrated Delivery Arena`
- 中文辅助：`智能体集成交付架构`

### 组合方式

| 场景 | 推荐组合 |
| --- | --- |
| PPT 封面 / 系统首页 | 图形标 + AIDA 字标 + 英文全称 |
| 系统导航栏 | 图形标 + AIDA |
| App 图标 / favicon | 仅使用图形标 |
| 汇报页脚 / 水印 | 单色图形标或 AIDA 字标 |
| 深色背景 | 白色图形标 + 红色路径 |

## 色彩建议（气质方向，不锁死具体配色）

> 界面具体配色**不预先定死**，交由 Claude Design 在气质护栏内多生成几版再择优。曾尝试"深炭底 + 大面积红"偏脏，应避免高饱和红大面积铺在深色底上。

气质方向：

- 整体维持企业级、可信、克制气质，红 = "要紧事"语义（品牌重点/风险）并保持克制。
- 不建议大面积蓝紫渐变；AIDA 的可信度来自结构，而不是特效。
- 红黑灰是其中一个可参考底色方向，但不作为唯一强制方案。

界面配色方向详见 `01_设计偏好文档/02_AIDA前端设计Brief_交付ClaudeDesign.md` 第 9 节（气质护栏）。

## 字体建议

### 英文字标

建议使用几何无衬线作为基础，并定制 `AIDA` 四个字母：

- `A`：顶部轻微开口或斜切，呼应智能体升级。
- `I`：保持极简竖向结构，承担“集成中轴”的语义。
- `D`：外轮廓保持稳重，不做过度圆润。
- 第二个 `A`：与首字母保持一致，形成首尾闭环。

可参考字体方向：`Inter`、`Aptos Display`、`Helvetica Now`、`DIN`、`HarmonyOS Sans`。正式商用前需要确认授权。

### 中文辅助

中文建议使用 `Microsoft YaHei`、`HarmonyOS Sans SC` 或企业已有字体。中文只做辅助，不进入主标核心图形，避免小尺寸失真。

## 图形规范

### 几何比例

- 图形标建议放入 `24 x 24` 或 `32 x 32` 网格。
- 左右斜柱保持对称，保证企业级稳定感。
- 红色连接梁应落在视觉重心略下方，避免图形显得头重脚轻。
- 中央负形通道必须清晰，保证小尺寸仍能读出“门 / 路径 / A”。

### 最小尺寸

| 用途 | 最小高度 |
| --- | --- |
| 图形标 | `20px` |
| 图形标 + AIDA | `28px` |
| 完整英文全称 | `48px` |

低于最小尺寸时，只保留图形标或 `AIDA`，不要强行放英文全称。

## 禁用方向

- 不使用机器人头、机械手、脑形、电路板、六边形蜂巢、盾牌、地球网格等泛 AI 图形。
- 不使用高饱和蓝紫渐变作为主识别。
- 不把 `AIDA` 做成花体、过度圆润或消费品牌风。
- 不在 Logo 内塞入复杂口号。
- 不用中文全称替代主标，中文只做辅助说明。

## 当前概念稿说明

本文件夹中的 `aida-logo-concept-primary.svg` 是第一版主推概念稿，用于内部评审方向，不是最终商标源文件。它体现了以下原则：

- 主体是可识别的大写 `A`。
- 红色路径表示集成连接与交付推进。
- 字标采用稳健的几何无衬线风格。
- 版式保留白底、深底、图标化三种使用空间。

## 下一步建议

1. 先内部确认是否接受 `AIDA Gateway` 作为主方向。
2. 如果接受，下一步输出三套矢量微调稿：稳健版、锋利版、轻量版。
3. 再确定标准色、黑白版、反白版、最小尺寸和 PPT / 系统导航栏应用规范。
4. 最后再进入正式 SVG / AI 源文件整理，避免直接把图像模型生成图当最终 Logo 使用。
