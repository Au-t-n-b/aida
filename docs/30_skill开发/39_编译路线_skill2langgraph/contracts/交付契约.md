> 关联：前门设计 §13；`SkillIR-schema.md`（流水线内部，非接缝）；`assumptions-schema.md`；`校验规则集.md`（Session 0 调用）。

## 1. 接缝定义

```text
┌─────────────────────┐   交付契约（唯一接缝）   ┌──────────────────────┐
│  引导编写前门         │  SKILL.md + assumptions.json │  编译流水线           │
│  · 独立程序/业务侧    │ ───────写文件──────────▶ │  · 开发者/CI 侧       │
│  · build-time HITL    │     → Session 0 闸门 →     │  · 可无头             │
└─────────────────────┘                          └──────────────────────┘
```

## 2. 产物（两件物证）

| 物证 | 规格 | 说明 |
|---|---|---|
| `SKILL.md` | 《编写规范》§22 十五节模板 | 业务契约本体（Raw Skill） |
| `assumptions.json` | `assumptions-schema.md` | 已确认假设清单（语义真源的第二件） |

> 语义真源 = `SKILL.md` + `assumptions.json`（草稿 + 已确认假设），二者缺一不可。

## 3. 移交方式

- **写文件**，不做进程内调用——两系统进程/部署解耦。
- 写出后 → 交 **Session 0 契约校验闸门**（跑 `校验规则集.md`，与前门 S8 **同源**；此处为复核）。
- 通过 → 进编译（会话 A–G）；不通过 → 打回。

## 4. 来源中立

流水线接受**任何来源**的合规 Raw Skill：
- 前门程序产出的；或
- 懂《编写规范》的开发者**手写**的。

→ 契约冻结后，**用手写 Raw Skill 即可驱动整条流水线**，不必等前门实现（实现路线图 Step 2/3 据此先行）。

## 5. 合规判定

"合规" ≡ 过 `校验规则集.md` 的全部 `error` 级规则（完整性 R-C + 一致性 R-K + 运行时契约 R-A）。`SKILL.md` 与 `assumptions.json` 的交叉一致（每处收紧在 assumptions 有据、可被 SkillIR 回指）由 R-K-08 保证。

## 6. 版本与可追溯

- 两件物证各带 `schema_version`；`assumptions.json` 带 `doclib_version`（接地版本）。
- 编译产物（SkillIR/Graph/报告）记录所基于的契约版本，支撑再验证（实现路线图 §5 P3）。
