## 0. 你为什么会收到这份需求

最终形态里 Manager 负责"鉴权 + 一用户一容器 + 目录映射"（`04` §2.2/§10）。我们的测试方案**复用同一套调度基建**：CI 要能按指定镜像 digest 拉起一个**一次性测试容器**、把生成的 skill 与 work_root 注入进去、用测试租户身份鉴权、跑完销毁。下列需求是 Manager 侧要为"测试"这一类调用提供的能力。

> 落地状态提示：`04` §3 标注 **Manager · 鉴权 ❌待建、容器调度 ❌待建**（当前 P0 单机无 Manager）。本文是**最终形态需求**，请在 Manager 建设时一并纳入"测试调用"这一等公民场景，避免事后补丁。

## 1. 需求清单

### R1 · 按镜像 digest 拉起一次性测试容器
**要求**：提供一个调度接口，输入 = **AIDA 镜像 digest** + 测试租户 + skill 包 + 输入数据，输出 = 容器访问 endpoint（`/agent/...`）+ 分配的 `runId`。容器为**一次性**：跑完或超时即回收。
**关键**：digest 由调用方指定，且能查询"当前生产 digest"以保证 test==prod（见需求文档 04 §R3）。
**验收**：同一 digest 重复拉起行为一致；容器在 TTL/完成后被回收，无残留。

### R2 · 注入面：skill 包注入 + work_root↔REST 桥接
**要求**：
- **skill 注入**：拉容器时让生成的 skill + 新工具可被加载并注册（skill 定义+代码归**镜像**，经 AIDA Agent §R3 选定的加载机制；**不经数据中心**——数据中心是文件云空间，不存 skill 定义）。
- **★work_root ↔ REST 桥接（关键接缝）**：AIDA 工具读写**本地路径** `work_root`（`doc_read_xlsx` 等用 `Path(path)`），而数据中心是 **REST 文件 API**。Manager 需承接二者桥接，二选一并文档化：
  - (a) **下载/上传式**：run 前用数据中心 `GET /files/.../download` 把输入拉到容器 `work_root/input/`，run 后用 `POST /files/upload` 把 `output/` 产物回传项目云空间；或
  - (b) **挂载式**：把数据中心存储 bind-mount 进容器（AIDA 04 §2.2"零拷贝目录映射"设想，但需数据中心存储可挂载）。
- **run 态本地化**：`work_root` 是容器内一次性目录（含 `input/runtime/output` + 容器内 run 态）；run 执行态（`state.json`/checkpoint）**留容器**，不回写数据中心（见总设计 §2.1）。
**验收**：容器内 `SkillContext.work_root` 可读到输入、产物能落到数据中心项目云空间；`GET /agent/skills` 能见到注入的 skill。
**出处**：`04` §2.2/§7；总设计 §2.1、§9 B12；数据中心 03 §R2。

### R3 · 测试 service tenant 鉴权
**要求**：鉴权层支持一个**专用测试 service tenant**（机器身份，非真实客户）。测试调用以该租户鉴权通过；资源（容器机时 / token）归账到该租户。测试租户**仅可见自己的测试项目**，不可触达任何真实客户项目（爆炸半径隔离）。
**验收**：① 测试租户能拉容器、跑 run；② 测试租户尝试访问真实项目被拒；③ 资源用量可按测试租户单独统计。
**出处**：`04` §2.2（鉴权）；总设计 §7。

### R4 · 测试 run 生命周期与隔离
**要求**：每次 CI 测试拉**一次性容器**（run 执行态在容器内 SqliteSaver，用完随容器销毁）。云空间侧的产物隔离**按测试项目/目录**实现（数据中心无 run 概念，见数据中心 §R3）——并发测试用独立测试项目或目录，避免产物互相覆盖。
**验收**：并行 N 个测试 run，容器内 run 态互不串扰、云空间产物互不覆盖；容器销毁后数据按策略清理（数据中心 §R5）。
**出处**：总设计 §2.1（存储分工）、§7。

### R5 · 并发隔离与配额
**要求**：为测试调用提供**并发上限 / 队列**与单容器资源配额，避免 CI 大批量验证打爆调度器；同租户内 run 不互相抢占数据目录。
**验收**：超过并发上限的请求排队而非失败；单容器 OOM/超时被隔离回收，不影响他容器。

### R6 · 容器就绪 / 超时 / 回收契约
**要求**：调度接口返回容器后，提供"就绪信号"（配合 AIDA Agent §R8 `/healthz`）；定义 run 级**超时**与**强制回收**策略；回收时保证 `state.json` 已落盘可被取证读取（或在回收前回传证据）。
**验收**：① 驱动能确定性地等到容器就绪；② 超时容器被回收且证据不丢。

## 2. 与其他模块的接缝

| 接缝 | 对端 | 说明 |
|---|---|---|
| skill 包注入方式 | AIDA Agent §R3 | Manager 用 Agent 选定的加载机制装入 skill（归镜像，非数据中心） |
| work_root ↔ REST 文件桥接 | 数据中心 §R2 | Manager 下载输入到 work_root / 上传产物回云空间（或挂载式）；见 R2 |
| 测试用户与种子项目 | 数据中心 §R1 | 测试用户身份与预置已审批测试项目由数据中心提供 |
| 镜像 digest 来源 | CI 镜像 §R1/§R3 | Manager 按 CI 提供的 digest 拉容器 |
