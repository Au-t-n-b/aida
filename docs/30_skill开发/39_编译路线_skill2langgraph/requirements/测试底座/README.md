## 共同约定（所有模块通用）

1. **镜像 digest 是唯一版本轴**：凡涉及"基于哪个 AIDA 版本"的地方，都用镜像 digest 标识，不用语义版本号、不用 commit message。测试与生产**必须同一 digest**。
2. **行为对齐，不对实现对齐**：需求只约束**可观察的边界行为**（如"工具失败返回 `Error:` 前缀字符串"），不约束内部调用链。AIDA 内部重构只要边界行为不变即不破坏本方案。运行时边界行为以 `docs/30_skill开发/31_手写规范/AIDA-RUNTIME-CONTRACT.md` 为权威源。
3. **测试租户严格隔离**：测试用专用 service tenant，**无真实联系人/收件人配置**，**不可见任何真实客户项目**。严禁用真实账户测试。
4. **一次性、可复现**：测试 run 用完即销毁；测试身份与种子数据烤进镜像，随 digest 完全可复现。
5. **机器可校验优先**：凡"生成物必须满足 X"的约定，都要有对应 lint / 校验脚本 + 稳定退出码，不靠人肉评审。
6. **诚实标注落地状态**：各需求文档按 ✅已落地 / ⚠️部分 / ❌待建 标注，对齐 AIDA `04_容器化部署与运行时架构.md` 的诚实风格。
