# 03 · 文档健康诊断（Doc Health Diagnosis）

> **这是什么**：对 AIDA 团队文档与开发者视图的一次**可用性 / 时效性**诊断，给出根因与整改路线。属治理回路的「体检报告」——区别于 [01 系统活层](01_system_architecture.md)（架构现状）、[02 边界图](02_module_boundaries.md)（模块契约），本文回答：**文档本身可信吗、会不会漂、开发者读得动吗。**
>
> **怎么用**：架构师据此跑 Workflow A 生成整改 TASK；维护者据「整改建议」逐条收口；新规范/文档评审时对照 §6 的约定。
>
> ⏳ **诊断性质（重要）**：本文是 **2026-06-08 的时点快照，不是常青契约**。§3–§4 的实测证据（节点数、文件计数、日期）会随整改过期，仅作"病灶存在性"的证据；**§5 根因 + §6 建议 + §7 约定才是耐用部分**。整改完成后应在版本表登记并刷新证据或归档本文。

---

## 1. 一句话判断

**治理"机器守得住的地方铁律森严，散文守不住的地方自由漂移"——契约层（builder↔lint↔派生物）不漂，散文层（47 篇手维护 MD）已被证明会漂。** 可用性的最大坑是：读者分不清哪句是**现状**、哪句是**目标**。

---

## 2. 先认强项（治理已经赢下的地方）

| 强项 | 证据 |
|------|------|
| 单一真相是真的 | 工程手册集中于 [`30_skill开发/31_手写规范/`](../../30_skill开发/31_手写规范/README.md)，明文"唯一真相"；历史上 `docs/范式附录/` 曾镜像 8 份手册，因"同一文档两份"负债已退役并归档 |
| 派生 + 守门元规范 | 一组守门 lint（清单见 [AGENTS.md](../../../AGENTS.md)）+ [SDUI 画廊](../../site/sdui-gallery.html) / [文档站](../../site/index.html) 新鲜度守门——**契约类资产改了不同步即 CI 红** |
| 读者分层 | `agent/docs/README` 每文档标"读者"列；[开发者地图](../../00_开发者地图.md) §1"我是谁先看哪" |
| 角色化协同 | 架构师/开发者 SOP + [02 边界图](02_module_boundaries.md) + A/B/C/D 工作流 |

> 结论：**不缺治理意识，缺的是把"派生+守门"从契约层延伸到散文层。**

---

## 3. 时效性诊断 · 散文层在可证明地漂移

### 3.1 铁证 —— 同一事实三份手抄、三个错值
SDUI 节点数量，三处独立硬编码（实测 2026-06-08）：

| 出处 | 当时写的 | 代码真值 |
|------|----------|----------|
| `docs/30_skill开发/31_手写规范/SDUI.md`（已修） | 22 | 26 |
| 两份接入规范（同事 2 天前版本，已修） | 24 | 26 |
| `agent/sdui/builder.py::SduiNode`（真相） | — | **26** |

**三份副本、三个不同的错。** 这是散文层无守门的必然——只要靠人手抄枚举，就一定漂。

### 3.2 守门覆盖的致命不对称

```
被新鲜度守门：  2 个派生产物（sdui-gallery.html · docs/site）
零守门、自由漂： 47 篇散文 MD（约 8700 行）
```

锁上了**派生物**，却让**源头散文**裸奔——而散文才是开发者真正在读的。

### 3.3 其它陈旧信号（实测 2026-06-08）

- **双源 SKILL.md 已失同步**：仓库 `skills/` **4** 份 vs 运行时 `~/.claude/skills/` **3** 份。有个 skill 的元数据没部署到运行时，而 `lint_skill_contract` 读运行时位置——埋雷。
- **硬编码状态/日期遍地**：[开发者地图](../../00_开发者地图.md) "状态速览 2026-06-04"（已数日前）。**"当前状态 / 完成度"这类章节是设计上就会陈旧的**，放进常青文档 = 定时炸弹。
- **占位/未实装高占比**：约 **20/47** 篇含 `占位/待补/未实装/目标态/🟡🔴`。`05/06/07` 三篇明标"占位，同事补"已数日。

## 4. 可用性诊断 · "现状 vs 目标"分不清是头号病灶

刚对齐的两份接入规范就是活样本：它们把一个**当前不存在的"系统设计相"**（page 分派 / 6 执行节点 / `dump_node_json` / LLM `intent.py`）写得像现状，真实 xtsj 只是个 2 命令 dispatch PoC。**照着写会全程踩空**——这比"缺文档"更危险，因为它制造**虚假信心**。

已用 `⏳目标态` 标记 + 顶部"落仓核对"横幅缓解，但暴露系统缺口：**全仓没有"现状/目标/占位"的统一可信度标注约定**，读者与 coding agent 无法 O(1) 判断一句话能不能信。

| 其它摩擦 | 证据 / 影响 |
|----------|------------|
| 入口认知负荷 | 开发者地图 174 行 + 47 篇约 8700 行——新人"第一站"本身是长文 |
| 文档站硬伤 | 内嵌全部 47 篇（含 `ux/品牌与草图/归档/…拆分说明.md` 碎屑）；**文档间链接在 SPA 里点不动**（hash 路由未改写）；无"每篇新鲜度"提示 |
| 占位稀释信号 | 近半文档带占位标记，真材料被"待补骨架"稀释，检索命中率虚高 |
| 机读性（文档也喂 coding agent） | 散文漂移 + 现状/目标混淆，会让 Cursor/CC 学到**过时契约**（照写 24 节点 / `dump_node_json`）；AIDA 重度依赖 agent 读文档干活，危害被放大 |

---

## 5. 根因 · 一条清晰的分界线

```
        机器可验证                      人肉维护
   ┌────────────────────┐      ┌──────────────────────────┐
   │ 契约 / 派生产物       │      │ 散文 / 状态 / 示例          │
   │ → lint 守门 → 不漂   │      │ → 无守门 → 必漂             │
   │ builder ↔ sdui.ts   │      │ 节点数 22/24/26            │
   │ SDUI 画廊 / 文档站    │      │ 接入规范描述不存在的代码      │
   └────────────────────┘      │ 状态速览数日前              │
                                │ SKILL.md 4 vs 3           │
                                └──────────────────────────┘
```

**左边已经赢了，右边还没开打——而开发者 80% 时间在读右边。**

## 6. 整改建议（按性价比排序 · 全部复用已有 derive+gate 范式）

| # | 优先级 | 动作 | 落点 / 已有样本 |
|---|--------|------|----------------|
| 1 | **P0** | **根除手抄枚举**：文档里"枚举代码事实"（节点表/端点表/step 表/命令表）一律删手抄、改指向生成源 | 样本：SDUI.md 的 22-节点表已换成指向 [`sdui-gallery.html`](../../site/sdui-gallery.html)。直接消灭"22/24/26"整类漂移 |
| 2 | **P0** | **可信度标注约定**：非现状内容必须标 `⏳目标态/占位/废弃`，**未标=承诺现状**；加 `lint_doc_status` 扫"声称现状但锚点代码不存在" | 样本：接入规范的 `⏳目标态` 横幅。新增 `agent/scripts/lint_doc_status.py` |
| 3 | **P1** | **状态自动化**：把"当前状态/完成度/状态速览"从常青文档剥离，改 `gen_status.py` 从 skill registry + git 派生状态看板 | 新增 `agent/scripts/gen_status.py`；喂进文档站首页 |
| 4 | **P1** | **消灭双源 SKILL.md**：`skill_md_path` 统一指向仓库 `skills/`，废 `~/.claude/skills/` 副本（或 CI 强制 diff） | [Skill 接入规范 §5 增强项](../../30_skill开发/31_手写规范/接入/接入-Skill与LangGraph接入规范.md) 早记的 P1；当前 4 vs 3 即代价 |
| 5 | **P2** | **文档站补强**：① 文档间链接改写为 `#doc-N` hash 路由；② 排除/折叠 `归档/` 碎屑；③ 每篇显示 git 最后修改时间做"新鲜度红绿灯"；④ 顶部放 #3 状态看板 | [`agent/scripts/gen_docs_site.py`](../../../agent/scripts/gen_docs_site.py) |
| 6 | **P2** | **占位治理**：20/47 占位文档——能填则填、不填移入显式 `docs/_wip/` 或加统一 badge，拉回信噪比 | 全 `docs/` |

## 7. 沉淀为团队约定（耐用部分）

1. **文档不许手抄代码事实**——枚举类一律指向生成源 / 契约文件（#1）。
2. **未标即承诺现状**——目标/占位/废弃必须显式标注（#2）。
3. **状态不手打**——完成度/进度/日期由脚本派生（#3）。
4. **单源**——元数据（SKILL.md）一个物理真相，副本靠生成 + 守门（#4）。

> 最高杠杆一步是 **#1**：本会话的 SDUI 画廊 + 26 节点修正 + 接入规范 `⏳目标态` 标注，已是这条路线的三个落地样本——**把它制度化即可**。
