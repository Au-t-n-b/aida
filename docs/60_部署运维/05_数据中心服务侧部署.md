## 2. 对接契约锚点（开发者当前可依赖）

- **读数据**：前端/Claw 一律以 `runs/<runId>/state.json` 为单次 run 唯一真相；产物清单以 `state.outputs` 为准，**不扫 `output/` 目录**。
- **写数据**：Claw 后端经 `SkillContext.work_root` 写 `input/ runtime/ output/ state.json`，零拷贝即时可见。
- **隔离边界**：跨 run / 跨租户**不可**互访；目录前缀即隔离边界。
