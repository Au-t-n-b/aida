# AIDA 服务器部署指南

> **适用场景**：在内网服务器上跑通 `aida/frontend`（前端） + `aida/agent`（LangGraph 后端，含智慧工勘 zhgk skill），供团队通过浏览器访问。
>
> **服务器约定**（按实际情况替换）：
> - 部署机 IP：`10.143.2.197`（ServerA）或 `10.143.2.198`（ServerB，实测也可）
> - aida/agent 端口：`7401`
> - aida/frontend 端口：`8080`（若被占用用 `8081`，下文统称 `<前端端口>`）
>
> **内网三个必知前提**（实测踩坑，先看）：
> 1. **npm 可能无法访问外网** → 在本地 Windows 上 `npm run build` 后把 `dist/` 传到服务器（见第三步）
> 2. **智谱 API 需走内网代理** → `agent/.env` 里加 `HTTP_PROXY=http://10.143.2.250:8088/`（企业代理，SSL 自动关闭验证）
> 3. **端口 8080 可能已被占用** → `lsof -i:8080` 确认，改用 `8081` 等空闲端口

```bash
# 确认 Python 版本 ≥ 3.10

# 确认 Node.js 版本 ≥ 18
node --version

# 确认 git
```

## 第一步：拉代码到服务器

```bash
# 选一个合适的部署目录，推荐 /opt/aida
mkdir -p /opt/aida
cd /opt/aida

# 从 GitHub 克隆（内网若不通外网，见下方「离线方式」）
git clone https://github.com/Au-t-n-b/aida.git
cd aida
```

**离线方式**（内网无法访问 GitHub）：
```bash
# 在可联网机器上（或用 Cursor 从本地 D:\aida 上传）
# 将整个 D:\aida 目录打包传到服务器：
scp -r D:/aida user@10.143.2.197:/opt/aida/aida

# 或用 rsync（推荐，排除 node_modules / .venv）：
rsync -avz --exclude='node_modules' --exclude='.venv' --exclude='agent/runtime' \
    D:/aida/ user@10.143.2.197:/opt/aida/aida/
```

## 第二步：部署后端（aida/agent）

### 2.1 创建 Python 虚拟环境并安装依赖

```bash
cd /opt/aida/aida

# 建 venv（必须在仓库根执行，不要 cd agent 后再建）
python3 -m venv agent/.venv

# 激活
source agent/.venv/bin/activate

# 升级 pip 并安装依赖
pip install -U pip
pip install -r agent/requirements.txt
```

验证依赖安装成功：
```bash
python -c "import fastapi, langgraph, openai; print('后端依赖 OK')"
```

### 2.2 部署 SKILL.md（A 层，必须）

```bash
# SKILL.md 运行时读 ~/.claude/skills/<name>/，必须部署否则 lint 报错
mkdir -p ~/.claude/skills
cp -r /opt/aida/aida/skills/. ~/.claude/skills/

# 验证
ls ~/.claude/skills/
# 应显示：_template  guihua  zhgk
```

### 2.3 配置环境变量

```bash
cd /opt/aida/aida
cp agent/.env.example agent/.env
```

用编辑器打开 `agent/.env`，**必填**这几项：

```bash
# 用 vim 或 nano 编辑
vim agent/.env
```

```ini
# ── 必填：智谱 BigModel API Key ──
ZHIPU_API_KEY=你的智谱api_key

# ── 必填：zhgk 工作区路径（底表文件放这里）──
ZHGK_ROOT=/srv/zhgk

# ── 内网必填：企业代理（服务器无法直连 open.bigmodel.cn 时）──
# 代理地址替换为你们内网实际的 HTTP 代理
HTTP_PROXY=http://10.143.2.250:8088/
HTTPS_PROXY=http://10.143.2.250:8088/
# 企业代理会做 MITM SSL 拦截，aida/agent 检测到代理时自动关闭 SSL 验证
# 若确认代理不拦截 SSL，可设 ZHIPU_SSL_VERIFY=true 强制开启

# ── 可选：Langfuse 评测可视化 ──
# LANGFUSE_PUBLIC_KEY=pk-lf-xxx
# LANGFUSE_SECRET_KEY=sk-lf-xxx
```

### 2.4 准备 zhgk 工作区（底表数据）

```bash
# 创建工作区目录结构
python /opt/aida/aida/agent/scripts/init_zhgk_workspace.py \
    --dest /srv/zhgk

# 验证目录结构已创建（v4）
ls /srv/zhgk/ProjectData/
# 应显示：Images  Input  Output  RunTime  Template
```

**将底表文件放入 `Template/` 目录**（向数据中心同事索取，或从本地传过来）：

```bash
# v4 需要以下文件（放入 Template/）：
# - 入场评估标准表.xlsx          ← 必须（filter_build 读底表）
# - 工勘常见高风险库.xlsx        ← 必须（report_gen 风险识别）
# - 新版项目工勘报告模板.docx    ← 可选（缺失时用内置模板）

# 从本地传文件（在本地机器执行）：
scp "D:/path/to/入场评估标准表.xlsx" user@10.143.2.197:/srv/zhgk/ProjectData/Template/
scp "D:/path/to/工勘常见高风险库.xlsx" user@10.143.2.197:/srv/zhgk/ProjectData/Template/
scp "D:/path/to/新版项目工勘报告模板.docx" user@10.143.2.197:/srv/zhgk/ProjectData/Template/

# 验证文件到位
ls /srv/zhgk/ProjectData/Template/
```

> ⚠️ Template/ 目录的文件是工勘必须的固定资产。缺文件时 skill 会在第一步触发 HITL（要求上传），这是**正常行为**，不是 bug。

### 2.5 启动后端

```bash
cd /opt/aida/aida
source agent/.venv/bin/activate

# 前台运行（调试用）
uvicorn agent.main:app --host 0.0.0.0 --port 7401

# 后台运行（生产推荐，日志写文件）
nohup uvicorn agent.main:app \
    --host 0.0.0.0 \
    --port 7401 \
    --workers 1 \
    > /var/log/aida-agent.log 2>&1 &

echo "aida/agent PID: $!"
```

> **注意**：`--host 0.0.0.0` 让服务器对外可访问；`--workers 1` 是必须的（LangGraph 使用进程内 asyncio，多 worker 会冲突）。

### 2.6 验证后端

```bash
# 健康检查（应返回 "ok": true）
curl http://127.0.0.1:7401/healthz

# 查看已注册 skill
curl http://127.0.0.1:7401/agent/skills

# 检查 zhgk 工作区状态
curl http://127.0.0.1:7401/healthz | python3 -m json.tool
```

正常输出示例：
```json
{
  "ok": true,
  "skills": {
    "zhgk": { "start_files": 4, "input_files": 0 }
  },
  "llm": { "configured": true, "model": "glm-4.5" }
}
```

如果 `"configured": false`，说明 `ZHIPU_API_KEY` 未生效，重启 uvicorn 后再试。

## 第三步：构建并部署前端（aida/frontend）

### 3.1 安装前端依赖

> ⚠️ **内网服务器通常无法访问 npm registry**，推荐在**本地 Windows 上构建**后将产物 `dist/` 传到服务器，跳过服务器侧的 `npm install`。
>
> - **方式 A（推荐）：本地构建 → 上传 `dist/`**（见 3.2 → 3.3 → 直接到 3.4）
> - **方式 B：服务器直接 `npm install`**（需服务器能访问 npm，或配好内网镜像）

**方式 B（服务器有 npm）：**
```bash
cd /opt/aida/aida/frontend
npm install
```

### 3.2 配置前端环境变量

```bash
# 创建生产环境配置
cat > /opt/aida/aida/frontend/.env << 'EOF'
# aida/agent 地址（当前服务器 IP + 7401 端口）
VITE_AGENT_BASE=http://10.143.2.197:7401

# ClawManager 地址（如果已部署，否则留空或删掉这行）
# VITE_CLAWMANAGER_BASE=http://10.143.2.197:8000
EOF
```

> `VITE_*` 是**编译期**变量，写入 .env 后需重新 build 才生效。

### 3.3 构建前端

**方式 A（推荐）：本地 Windows 构建，产物上传服务器**

```powershell
# 在本地 D:\aida 执行
cd D:\aida\frontend

# 先写好 .env（指向服务器 IP）
@"
VITE_AGENT_BASE=http://10.143.2.198:7401
VITE_CLAWMANAGER_BASE=http://10.143.2.197:8000
"@ | Out-File -Encoding utf8 .env

# 构建
npm run build

# 上传 dist/ 到服务器（用 scp 或 rsync）
scp -r dist/ user@10.143.2.198:/opt/aida/aida/frontend/dist/
# 或用 rsync：
# rsync -avz dist/ user@10.143.2.198:/opt/aida/aida/frontend/dist/
```

**方式 B（服务器有 npm）：服务器上直接构建**

```bash
cd /opt/aida/aida/frontend
npm run build
```

构建成功后，产物在 `frontend/dist/` 目录。

### 3.4 启动前端静态服务

```bash
# 方式 A：用 Python 简易服务（调试足够用）
cd /opt/aida/aida/frontend/dist
nohup python3 -m http.server 8080 > /var/log/aida-frontend.log 2>&1 &
echo "前端 PID: $!"

# 方式 B：用 vite preview（等同于生产静态托管）
cd /opt/aida/aida/frontend
nohup npm run preview -- --host 0.0.0.0 --port 8080 \
    > /var/log/aida-frontend.log 2>&1 &
echo "前端 PID: $!"
```

### 3.5 验证前端

```bash
curl -s http://127.0.0.1:8080 | grep -o '<title>.*</title>'
# 应输出 <title>AIDA</title> 之类的标题
```

浏览器访问：`http://10.143.2.197:8080`

## 第五步：设置开机自启（可选）

```bash
# 创建 systemd 服务文件
cat > /etc/systemd/system/aida-agent.service << 'EOF'
[Unit]
Description=AIDA Agent (LangGraph)
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/aida/aida
ExecStart=/opt/aida/aida/agent/.venv/bin/uvicorn agent.main:app --host 0.0.0.0 --port 7401 --workers 1
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

cat > /etc/systemd/system/aida-frontend.service << 'EOF'
[Unit]
Description=AIDA Frontend (Static)
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/aida/aida/frontend/dist
ExecStart=/usr/bin/python3 -m http.server 8080
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

# 启用并启动
systemctl daemon-reload
systemctl enable aida-agent aida-frontend
systemctl start aida-agent aida-frontend

# 查看状态
systemctl status aida-agent
systemctl status aida-frontend
```

## 常见问题排查

| 现象 | 排查方向 |
|------|---------|
| `healthz` 返回 `"configured": false` | `agent/.env` 的 `ZHIPU_API_KEY` 未生效，重启 uvicorn |
| 聊天返回 `Connection error` / `401 令牌过期` | 智谱 API Key 失效，登录 bigmodel.cn 重新生成 |
| 聊天返回 `Connection error`（Key 正确） | 服务器无法直连智谱 API，在 `agent/.env` 加 `HTTP_PROXY=http://10.143.2.250:8088/` 后重启 |
| `ModuleNotFoundError: No module named 'agent'` | uvicorn 不在仓库根 `/opt/aida/aida/` 执行，检查 `WorkingDirectory` |
| `npm install` 卡住或超时 | 服务器无法访问 npm registry，改用「本地构建 → 上传 dist/」方式 |
| 前端请求打到 `127.0.0.1:7401`（自己电脑） | `frontend/.env` 没有设 `VITE_AGENT_BASE`，或设了但未重新 build |
| 启动工勘立即 HITL | `Start/` 目录缺底表文件，上传后点继续（正常行为） |
| 前端白屏 | 检查 `.env` 是否在 `npm run build` 前创建；检查静态服务是否在 `dist/` 目录下启动 |
| `git pull` 后前端未更新 | 需要重新 `npm run build` 并重启静态服务 |
| 端口 8080 被占 | `lsof -i:8080` 确认占用进程，改用其他端口（如 `8081`）启动静态服务 |
| `python3-venv` 不存在 | `apt install python3.10-venv` 或对应 Python 版本的 venv 包 |
| `/etc/hosts` 无法解析 bigmodel.cn | 确认代理已配置；或在 `/etc/hosts` 追加 IP（需先 `nslookup` 查真实 IP） |

## 附录 A：服务器环境安装

### Python 3.10+（CentOS/RHEL）
```bash
# 方式1：yum（如果有 epel 源）
yum install python310

# 方式2：编译安装（内网无源时）
wget https://www.python.org/ftp/python/3.11.9/Python-3.11.9.tgz
tar xzf Python-3.11.9.tgz && cd Python-3.11.9
./configure --enable-optimizations && make -j4 && make install
```

### Python 3.10+（Ubuntu/Debian）
```bash
apt update && apt install -y python3.11 python3.11-venv python3-pip
```

### Node.js 20（推荐用 nvm）
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
```

## 附录 B：更新部署

代码有更新时，执行以下步骤：

```bash
cd /opt/aida/aida

# 1. 拉最新代码
git pull

# 或从本地同步（本地有修改未 push 时）：
rsync -avz --exclude='node_modules' --exclude='.venv' --exclude='agent/runtime' \
    D:/aida/ user@10.143.2.197:/opt/aida/aida/

# 2. 更新后端依赖（如 requirements.txt 有变）
source agent/.venv/bin/activate
pip install -r agent/requirements.txt

# 3. 重启后端
systemctl restart aida-agent
# 或：pkill -f "uvicorn agent.main" && nohup uvicorn ... &

# 4. 重建前端（如前端代码或 .env 有变）
cd frontend
npm run build

# 5. 重启前端服务（如用 systemd）
systemctl restart aida-frontend
```
