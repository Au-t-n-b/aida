## 2. 对接契约锚点（Manager 必须满足，已被约束死）

- **目录映射契约**（硬接口，见 04 §7）：拉容器时把 `projects/<pid>/skills/<skill>/runs/<runId>/` 挂为 Claw 的 `work_root`，并挂 `skills/<skill>/`（只读资产）。容器**只能见自己租户的目录**。
- **运行 ID 约定**：每次作业一个 `runId`，目录三级隔离根除并发覆盖（数据中心 README 审计 P0）。
- **与后端解耦**：Claw 内的 LangGraph **不感知** Manager —— work_root 由注入决定，Claw 不写死前端地址、不写死租户路径（这样单机/容器两种形态共用一套 Claw 代码）。
