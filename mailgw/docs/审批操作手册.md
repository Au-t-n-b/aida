# mailgw 邮件网关 · 审批操作手册

> **读者**：负责审批 agent 外发邮件的审批人
> **版本**：v1.0（2026-06-10）

## 1. 审批页是做什么的

AIDA 等智能体（agent）通过邮件网关对外发送邮件。为防止发错对象或内容不当：

- 收件人**全部在白名单内**（如公司域名邮箱）的邮件 → 自动直接发出，无需审批；
- 收件人**含任何白名单外地址**的邮件 → 暂存到待审批队列，**经你确认后才会真正发出**。

## 2. 登录

1. 浏览器打开 `http://<网关地址>:8025/admin`（具体地址与端口以管理员提供为准）；
2. 弹出登录框：用户名固定为 `admin`，口令由管理员设置（`.env` 中的 `MAILGW_ADMIN_PASSWORD`）。

## 3. 审批操作

页面上半部分是**待审批列表**，各列含义：

| 列 | 含义 |
|---|---|
| 提交时间 | agent 提交该邮件的时间（UTC） |
| 调用方 | 哪个 agent 提交的（如 `aida`） |
| 收件人 | 全部收件人与抄送地址 |
| 主题 | 邮件主题 |
| 正文预览 | 正文前 500 字 |
| 附件 | 附件文件路径清单 |

操作：

- **通过**：点击后邮件**立即发出**，发送结果可在"最近处理"中看到；
- **驳回**：必须先在输入框填写驳回原因，再点驳回。**原因会原样返回给 agent**（agent 可能转述给最终用户），请写明确、可执行的修改建议，例如"收件人公司外域名未经报备，请改为内部联系人"。

## 4. 审批要点（检查清单）

逐项确认后再点"通过"：

1. **收件人**：地址是否正确？该外部对象是否确实需要收到此邮件？
2. **正文**：是否包含不应外发的敏感信息（报价、内部数据、个人信息）？表述是否得体？
3. **附件**：文件名/路径看起来是否为预期交付物（如工勘报告），有没有夹带无关文件？
4. **意图**：结合调用方与主题判断这次外发是否符合业务流程；可疑时驳回并注明原因。

> 提示：agent 的邮件内容由大模型生成，可能存在事实错误（幻觉）。对外承诺类内容（时间、价格、责任）尤其要人工核对。

## 5. 追溯

- 页面下半部分"最近处理（50 条）"显示最近的任务及状态（`sent` 已发出 / `rejected` 已驳回 / `failed` 发送失败）与说明；
- 更早的完整操作记录保存在网关数据库的 `audit_log` 表（含每次提交、审批、驳回的时间与操作人），需要时联系管理员导出查询。
